# Agent-Harness Doc Principles

Apply these as tests while refactoring docs.

## Source of truth

- The repo is the source of truth. Knowledge outside it is invisible until it is
  encoded as markdown, code, schemas, tests, generated docs, or plans.
- `AGENTS.md` is a map, not a manual.
- Enforce the tree in `architecture-template.md`.
- Keep generated docs tied to their source and regeneration command.
- Track durable decisions and accepted debt in docs, not chat.
- Enforce no-comments code policy. Keep only verified functional directives.

## Writing standard

- Short and linked beats long and self-contained.
- One fact has one canonical owner.
- Replace repeated prose with links.
- Make hidden knowledge visible in the repo.
- Enforce invariants mechanically with lints, tests, generated checks, or CI.
- Remove comments generated by agents unless they are required by the language or
  toolchain.
- Do not invent facts to fill a template. State unknown, not applicable, or link
  to the owner.

## Refactor checklist

- [ ] Required docs tree exists.
- [ ] File owns a clear topic.
- [ ] Repeated facts moved to one owner.
- [ ] Filler, hedging, and restatement removed.
- [ ] Links resolve.
- [ ] One H1.
- [ ] Heading levels do not skip.
- [ ] Commands and code blocks preserve behavior.
- [ ] Generated docs name their source and regeneration command.
- [ ] Code comments are absent except verified functional directives.
- [ ] Accepted debt is recorded in `docs/exec-plans/tech-debt-tracker.md`.
